# 低光照图像增强：基于双边滤波与自适应引导滤波的 Retinex 对比研究

## 📖 项目简介

本项目是数字图像处理课程的低光照图像增强实验。低光照图像通常存在亮度不足、局部细节不可见、严重噪声以及对比度较低等问题 。为了解决传统直方图均衡化和全局 Gamma 变换带来的局部过曝和高光细节丢失问题，本项目回归经典的 **Retinex 物理成像模型** 。

本项目构建了一套严格控制变量的低光照图像增强实验框架 。通过将亮度补偿参数严格恒定（$\gamma=0.6$），绝对公平地量化对比了底层视觉领域两大核心平滑算子——**双边滤波器 (Bilateral Filter, BF)** 与 **自适应引导滤波器 (Adaptive Guided Filter, AGF)** 在光照图估计与“保边平滑”中的真实物理表现 。

### ✨ 核心创新点

针对传统 Retinex 方法的痛点，本项目提出了两项核心创新机制：

1. **自适应引导正则化 (Adaptive Guided Regularization):**
   [cite_start]引入基于局部方差的自适应调整策略，赋予滤波器动态感知纹理与边缘的能力 。在强物理边缘处施加严厉的“保边”约束，在平坦区域退化为强力平滑，从而根本上遏制了强对比边缘的梯度反转与“光晕 (Halo)”伪影 。同时保持了极低且恒定的 $O(N)$ 计算复杂度 [cite: 10]。
2. **基于照明权重的抗噪融合策略 (Illumination-aware NLM):**
   [cite_start]设计了动态融合掩码 $W_{noise} = 1 - T_{refined}^{\gamma}$，将局部的“降噪强度”与“提亮幅度”进行直接强绑定。在暗部进行雷霆除噪的同时，完美保留了高光区的高频真实纹理，有效解决了由于除法提亮导致的严重噪声放大效应 。

## 🛠️ 环境依赖与安装

本项目依赖深度学习感知模型（LPIPS）及传统的图像处理库。请按照以下步骤在本地配置运行环境：

### 安装依赖包

确保环境已激活，然后安装 `requirements.txt` 中列出的依赖：

PowerShell

```
pip install -r requirements.txt
```

*(主要包含：`opencv-python`, `numpy`, `torch`, `lpips`, `scikit-image`)*

## 🚀 运行指南

在项目根目录下，确保虚拟环境处于激活状态，直接运行 `src` 目录下的主程序：

PowerShell

```
python src/main.py
```

**程序运行流程说明：**

1. 自动加载 LPIPS 视觉感知评价深度学习模型。
2. 遍历 `samples/Train/Input/` 目录下的所有低光照图像。
3. 分别使用 **双边滤波 (BF)** 和 **自适应引导滤波 (GF)** 进行光照解耦与增强处理。
4. 自动计算 PSNR, SSIM, MSE, MAE, LPIPS, SAM, LOE, NCC 等 8 项专业画质评估指标。
5. 将最终的增强图像保存至 `results/final/`，将过程光照图保存至 `results/intermediate/`。
6. 在控制台输出每张图像的评估结果，并在处理完成后打印**全局总评对比表**。

## 📊 实验结果摘要

在严格统一 $\gamma=0.6$ 补偿以及统一 NLM 降噪模块的绝对公平基准下 ，实验取得了以下显著结论：  

| **评价指标** | **指标倾向**       | **双边滤波 (BF)** | **自适应引导滤波 (GF)** | **性能对比**                    |
| ------------ | ------------------ | ----------------- | ----------------------- | ------------------------------- |
| **PSNR**     | ↑ 更高越好         | 12.73             | **12.84**               | GF 领先 (+0.11)                 |
| **SSIM**     | ↑ 更高越好         | 0.7220            | **0.7267**              | GF 领先 (+0.0047)               |
| **MSE**      | ↓ 更低越好         | 3883.12           | **3802.59**             | GF 领先 (-80.83)                |
| **MAE**      | ↓ 更低越好         | 0.22              | 0.22                    | **绝对持平** (印证控制变量成功) |
| **LPIPS**    | ↓ 感知失真越低越好 | 0.2030            | **0.1951**              | GF 领先 (-0.0079)               |
| **SAM**      | ↓ 色彩偏离越小越好 | 0.1184            | **0.1183**              | GF 领先 (-0.0001)               |

**结论分析：**

**消除伪影，提升结构保真度：** GF 在反映图像结构和感知的核心指标（PSNR, SSIM, MSE, LPIPS）上全面优于 BF 。特别是高度契合人类主观视觉的 LPIPS 指标显著下降，客观证实了自适应机制有效消除了“边缘光晕 (Halo)”现象 。  

**控制变量的严谨性：** 两种算法的 MAE（平均绝对误差）精确持平于 0.22，证明了全局亮度拉伸幅度完全相等，所有指标差异 100% 来源于平滑算子对光照图边缘的处理优劣 。  

**工程落地优势：** AGF 的时间复杂度严格保持在 $O(N)$，且计算速度与滤波窗口半径的大小完全无关，极大提升了在算力受限设备上的落地价值 。
